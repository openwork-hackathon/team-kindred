# Arcade Token Design — Kindred MVP

**Last Updated:** 2026-02-08  
**Status:** BRAINSTORM → MVP SPEC  
**Owner:** JhiNResH

---

## 1. Executive Summary

Kindred 採用**二層 Token 模型**：

| Phase | Token | 目的 | 盈利 |
|-------|-------|------|------|
| Phase 1 (Now-2028) | Arcade Token | 忠誠度計畫，評論者獲利 | Hook 手續費增加 |
| Phase 2 (2028+) | Network Token | 社區治理 + 分紅 | Token 升值 + DAO 分紅 |

**MVP 目標：** 設計完整的 Arcade Token Faucet/Sink 機制，使評論者通過「信譽 → Token → 交易折扣」的飛輪自動運作。

## 2. Arcade Token 基本參數

### 2.1 名稱 & 供應
- **名稱:** `$ARC` (Arcade Token)
- **初始供應:** 無上限（按需發行）
- **不可交易:** ✅ 只能在 Kindred 生態內使用
- **存儲:** Base Sepolia / Base Mainnet

### 2.2 Token 值與定價

**Faucet 價格（買入天花板）**
```
1 USD = 100 ARC
（固定，不變）
```

**Sink 價格（兌現底線）**
```
評論消耗: 1 ARC
交易手續費: ARC 支付（根據等級折扣）
高級功能: X ARC
```

**Buyback 機制（可選）**
```
平台每月回購：年度 Hook 手續費的 5% 用於 ARC 回購
回購價格: 0.8 USD / 100 ARC (20% 折扣)
```

## 3. Token 獲取 (Faucet) 機制

### 3.1 初期 Bootstrap (前 3 個月)

**登入獎勵**
```
首次登入: 100 ARC（一次性）
每日登入: 10 ARC（連續 7 天）
```

**任務系統**
| 任務 | 獎勵 | 說明 |
|------|------|------|
| 完成 KYC/GPS | 50 ARC | 一次性，驗證身份 |
| 發第一條評論 | 20 ARC | 一次性 |
| 評論被 upvote | 1 ARC/upvote | 上限 10 ARC/天 |
| 邀請朋友 | 50 ARC | 一次性，被邀請人註冊時 |

**平台補貼**
- 初期三個月，平台每月補貼 10,000 ARC 用於獎勵
- 用完後轉向「信譽等級產出」

### 3.2 長期機制 (3 個月後) — 等級自動產出

**信譽等級與每日產出**
```
等級 1: 0 ARC/天    (新用戶，沒有評論)
等級 2: 5 ARC/天    (1-5 條評論，信譽 >50%)
等級 3: 10 ARC/天   (6-20 條評論，信譽 >70%)
等級 4: 20 ARC/天   (21-50 條評論，信譽 >80%)
等級 5: 50 ARC/天   (51-100 條評論，信譽 >90%)
等級 6: 100 ARC/天  (>100 條評論，信譽 >95%)
```

**信譽計算（來自 on-chain 行為）**
```
基礎信譽 = (被 upvote 的評論數 / 總評論數) × 100%

修正因子：
- 被異議且敗訴 → -20% 信譽
- 無活動 30 天 → -5% 信譽/周
- 被社區舉報詐欺 → 降級 2 級
```

**獲得方式**
- 自動每日發放（錢包轉賬）
- 基於前一天的等級和信譽
- 無需做任何操作

## 4. Token 消耗 (Sink) 機制

### 4.1 評論成本

```
發一條評論 = 1 ARC 燃燒

例外：
- 等級 5+: 評論成本 0.5 ARC
- 等級 6: 評論免費（0 ARC）
```

**為什麼燃燒？**
- 防止垃圾評論（有成本壓制噪音）
- 維持 Token 稀缺性（供需平衡）
- 自動通脹控制

### 4.2 交易手續費折扣

**標準 Hook 手續費**: 0.5% (UNI 標準)

**根據等級調整**
```
等級 1: 0.8% (不使用評論系統)
等級 2: 0.65% 
等級 3: 0.55%
等級 4: 0.45%
等級 5: 0.35%
等級 6: 0.20%

支付方式：可用 ARC 或 ETH/USDC
- 如果 ARC 餘額足夠: 優先用 ARC（評論系統的人獲利）
- 如果 ARC 不足: 用 ETH/USDC 補差
```

**例子：等級 6 用戶交易**
```
交易 1000 USDC → UNI
- 標準費用: 0.5% = 5 USDC
- 等級 6 折扣: 0.20% = 2 USDC
- 節省: 3 USDC

如果用 ARC:
- 2 USDC @ 1 USD = 100 ARC → 200 ARC
- 1 條評論成本 = 1 ARC
- ROI: 200 次交易就能用一條評論成本換 0.2% 的手續費優惠
```

### 4.3 高級功能（未來）

```
置頂評論: 10 ARC/周
認證徽章: 50 ARC/月
私密評論: 5 ARC/條
```

## 5. Token 流動與平衡

### 5.1 供應流量預測 (假設 10,000 用戶)

**每日新增 ARC**
```
等級分佈假設:
- 等級 1: 30% × 0 = 0 ARC
- 等級 2: 40% × 5 = 2 ARC
- 等級 3: 20% × 10 = 2 ARC
- 等級 4: 7% × 20 = 1.4 ARC
- 等級 5: 2.5% × 50 = 1.25 ARC
- 等級 6: 0.5% × 100 = 0.5 ARC

每用戶平均: 6.75 ARC/天
全平台: 67,500 ARC/天
```

**每日消耗 ARC**
```
假設:
- 30% 用戶發評論: 3,000 人
- 平均 2 條/人/周 = 0.3 條/人/天

評論消耗: 3,000 × 0.3 × 0.8 ARC (平均等級) = 720 ARC/天

交易消耗 (可選):
- 如果 50% 用戶每周交易 1 次 = 5,000 人
- 平均 1 筆 $5,000 交易 
- 費用: 25 USD × 2 = 50 ARC

總消耗: ~800 ARC/天
```

**淨流動**
```
供應: 67,500 ARC/天
消耗: 800 ARC/天
淨增: 66,700 ARC/天 ⚠️ 嚴重通脹！
```

**✅ 調整方案**

| 調整 | 新供應 | 效果 |
|------|---------|------|
| 降低每日產出 50% | 33,750 | 淨增 32,950 (仍多) |
| 增加評論消耗到 2 ARC | 67,500 | 消耗 1,600 → 淨增 65,900 (仍多) |
| 添加交易 sink (強制用 ARC 支付 10% 手續費) | 67,500 | 消耗 5,000 → 淨增 62,500 (仍多) |
| **三者結合** | 33,750 | 消耗 2,400 → **淨增 31,350** (可控) |

**→ 建議 MVP 參數調整：**
```
等級產出 -50%:
等級 6: 50 ARC/天 (from 100)
等級 5: 25 ARC/天 (from 50)
...

評論消耗: 2 ARC (from 1)

交易手續費中，10% 強制用 ARC 支付
```

### 5.2 穩定性檢查

**6 個月用戶增長預測**
```
Month 1: 1,000 users → 33,750 ARC/天
Month 2: 2,000 users → 67,500 ARC/天
Month 3: 5,000 users → 168,750 ARC/天
Month 6: 50,000 users → 1,687,500 ARC/天
```

**如果用戶超過預期...**
- 自動降低每日產出比率（governor function）
- 或提高評論消耗
- 或調高交易手續費中 ARC 支付的比例

## 6. 防作弊機制

### 6.1 評論驗證

**Proof of Actually Used (GPS + On-Chain)**
```
評論發起者必須：
1. GPS 位置在餐廳 500m 內
2. 該錢包有該餐廳的消費記錄（可選）
3. 24 小時後才能評論（冷卻期）
```

**Dispute Mechanism**
```
任何人可以異議評論：
- 異議發起者質押: 5 ARC
- 被異議者質押: 5 ARC
- 社區投票 (等級 3+ 用戶)
- 勝方返還質押 + 獲得 5 ARC 獎勵
- 敗方燃燒質押 + 信譽 -20%
```

### 6.2 信譽衰減

```
無活動 30 天 → 等級 -1
無活動 60 天 → 等級 -2
無活動 90 天 → 重置為等級 1
```

## 9. 盈利模型確認

### Phase 1 (Arcade Token 時期) - 2026-2028

**收入來源**
1. **Hook 手續費增加** (主要)
   - Uniswap Hook: 0.5% base fee
   - 流量增加 → Fee pool 增加
   - 假設: 年交易量 $100M → Hook fees: $500k
   
2. **訂閱制** (後期)
   - Premium 功能: $9.99/月
   - 假設: 10% 用戶訂閱 → $1M/年

3. **廣告** (可選)
   - 餐廳在平台廣告
   - 或協議在 Kindred 宣傳

**支出**
- 初期 Token 補貼: $100k
- 運維: $50k/月

### Phase 2 (Network Token 時期) - 2028+

**收入轉變**
- DAO 金庫分成 Uniswap fee pool
- $KIND token 升值（市場估值）
- Kindred 公司作為核心貢獻者獲得分紅

## 10. 風險 & 緩解

| 風險 | 影響 | 緩解方案 |
|------|------|--------|
| Token 通脹失控 | 評論者無激勵 | 動態調整產出率，設置 hardcap |
| 評論品質下降 | 平台信譽受損 | Dispute + 信譽衰減 |
| 用戶不交易 | Hook fees 無法增長 | 其他收入源（訂閱、廣告） |
| 安全漏洞 | Token 被盜刷 | 審計 + 漏洞賞金計畫 |
| 監管風險 | ARC 被歸類為證券 | a16z 文章作為法律參考 + 無 buyback |
